# Git & GitHub

Die Versionsverwaltung und Team-Kollaboration erfolgt über **Git** und wird auf **GitHub** gehostet.

### Workflow
- **Repository:** Das Projekt wird zentral auf GitHub verwaltet.
- **Kollaboration:** Alle Teammitglieder haben Zugriff auf das Repository und arbeiten gemeinsam an den Features.
- **Änderungen:** Code-Änderungen werden mittels `git push` und `git pull` synchronisiert.
- **Merge-Management:** Wir nutzen Pull-Requests und Merges, um Features in den Hauptzweig zu integrieren. Merge-Konflikte werden lokal in der IDE oder im Terminal aufgelöst, bevor sie final committet werden.

Durch diesen strukturierten Workflow wird sichergestellt, dass der `master`-Branch stets stabil bleibt und alle Änderungen nachvollziehbar sind.
